# 3. Regular Expression Matching

## 題目
實作正則表達式匹配,支援 '.'(匹配任意單一字元)和 '*'(匹配零個或多個前面的元素)。

## 面試重點
此題是**複雜 DP 問題**,測試對狀態空間建模與邊界條件處理的深度理解。

核心思路:
- 狀態定義:dp[i][j] = s[0:i] 是否匹配 p[0:j]
- 狀態轉移:
  - 若 p[j-1] != '*':
    - 若 s[i-1] == p[j-1] 或 p[j-1] == '.':dp[i][j] = dp[i-1][j-1]
  - 若 p[j-1] == '*':
    - 匹配 0 次:dp[i][j] = dp[i][j-2]
    - 匹配 1+ 次:若 s[i-1] == p[j-2] 或 p[j-2] == '.':dp[i][j] = dp[i-1][j]

面試官會重點觀察:
- 能否正確處理 '*' 的兩種情況:匹配 0 次和多次
- 邊界初始化:dp[0][0] = true,以及 p 開頭為 a* 類型的處理
- 索引對應關係:dp 陣列與字串索引的錯位
- '*' 與前一個字元的綁定關係理解
- 能否解釋為何是 dp[i-1][j] 而非 dp[i-1][j-2]
- 遞迴解法與 DP 解法的對應關係

**常考公司**:
- Google(系統設計時可能涉及搜尋引擎)
- Microsoft(文字處理相關產品)
- 編譯器/解析器相關公司
- 高頻面試題,考察 DP 功底

此題是**字串 DP 的終極挑戰**,涉及多重條件判斷與狀態轉移。

## 複雜度
- 時間複雜度:O(m × n)
- 空間複雜度:O(m × n),可優化至 O(n)
