# 17. Implement Trie (Prefix Tree)

## 題目
實作 Trie(前綴樹),支援以下操作:
- insert(word):插入單詞
- search(word):查詢單詞是否存在
- startsWith(prefix):查詢是否有單詞以該前綴開頭

## 面試重點
此題測試**Trie 的基本實作**,是字串處理的核心資料結構。

核心設計:
1. **TrieNode 類別**:
   - children:HashMap 或陣列(26 個字母)
   - isEnd:標記是否為單詞結尾
2. **insert**:
   - 逐字元遍歷,不存在則創建新節點
   - 最後節點標記 isEnd = True
3. **search**:
   - 逐字元遍歷,不存在則回傳 False
   - 到達最後檢查 isEnd
4. **startsWith**:
   - 逐字元遍歷,能走完即為 True

面試官會重點觀察:
- TrieNode 設計:children 用 HashMap vs 陣列
- 陣列優化:若只有小寫字母,用 children[26]
- isEnd 的作用:區分前綴與完整單詞
- 空間時間權衡:Trie 空間大但查詢快
- Trie 的應用場景:自動補全、拼寫檢查、IP路由

**常考公司**:
- 所有公司的基礎題
- Google、Microsoft(字串處理)
- 搜尋引擎相關
- 通常作為其他題目的子結構

此題是**Trie 的標準模板**,必須熟練掌握。

## 複雜度
- insert:O(m),m 為單詞長度
- search:O(m)
- startsWith:O(m)
- 空間複雜度:O(ALPHABET_SIZE × N × M),N 為單詞數,M 為平均長度
