

# 52. Minimum Window Substring

## 題目
給定字串 s 和 t,找出 s 中包含 t 所有字元的最小子字串。若不存在則回傳空字串。

## 面試重點
此題是**滑動窗口的終極應用**,測試對窗口伸縮條件的精確控制。

核心思路:
1. 用 HashMap 記錄 t 中每個字元的需求次數
2. 雙指標維護窗口:
   - right 擴展窗口,減少對應字元的需求
   - 當窗口包含所有字元時,left 收縮窗口
   - 記錄最小窗口
3. 用 required 變數追蹤還需要多少不同字元
4. 用 formed 變數追蹤當前窗口已滿足多少字元

面試官會重點觀察:
- 能否正確處理字元重複:需要計數而非簡單的 Set
- 窗口有效性判斷:formed == required
- 何時收縮窗口:找到有效窗口後
- 如何更新最小窗口:記錄起始位置和長度
- 優化:用陣列替代 HashMap(若限定字元集)
- 邊界:t 為空、s 中不包含 t 的所有字元

**常考公司**:
- Facebook/Meta(滑動窗口高頻)
- Google、Amazon
- 字串處理相關公司
- 文本分析、搜尋引擎

此題展現了**滑動窗口在複雜約束下的應用**,是字串問題的巔峰。

## 複雜度
- 時間複雜度:O(|S| + |T|)
- 空間複雜度:O(|S| + |T|)
